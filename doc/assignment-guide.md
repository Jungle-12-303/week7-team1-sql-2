# B+ 트리 인덱스 구현 과제 가이드

## 1. 목적

- 단 하루 동안 AI를 적극적으로 활용하여 팀별 프로젝트에 집중적으로 몰입합니다.
- 결과물뿐 아니라 구현 과정 또한 중요합니다. AI가 작성한 코드라도 팀원 스스로 충분히 이해하고, 핵심 원리를 설명할 수 있어야 합니다.
- 익숙하지 않은 기능도 Top-down 방식으로 빠르게 분석하고 구현하며, 학습 민첩성과 실무 감각을 함께 향상시키는 것을 목표로 합니다.

## 2. 요구사항

- 이번 주 과제는 `B+ 트리 인덱스` 구현입니다.
- 실제 DBMS에서 사용하는 디스크 기반 B+ Tree를 단순화하여, 메모리 기반 방식으로 구현합니다.
- 테이블에 레코드가 추가되면 ID가 자동으로 부여되어야 하며, 해당 ID를 B+ 트리에 인덱스로 등록해야 합니다.
- 이후 `WHERE id = ?` 형태의 ID 기반 검색 시 B+ 트리 인덱스를 활용할 수 있어야 합니다.
- 대규모 데이터 삽입 이후 성능 테스트를 수행해야 합니다.
- 최소 `1,000,000`개 이상의 레코드를 Insert합니다.
- ID 기준 `SELECT`(인덱스 사용)와 다른 필드 기준 `SELECT`(선형 탐색) 간의 속도 차이를 비교합니다.
- 이전 차수에서 구현한 SQL 처리기를 그대로 활용하여 연동합니다.
- 구현 언어는 C 언어로 제한합니다.
- 이번 과제는 학습보다 구현 완성이 우선입니다. AI를 적극적으로 활용하여 결과물을 만들어내는 것이 가장 중요합니다.
- 다만, 생성된 결과물 중 핵심 로직은 반드시 이해하고 설명할 수 있어야 합니다.
- 직접 코드를 작성하기 어려운 경우 전체 코드를 AI로 생성해도 무방하지만, 생성된 코드는 반드시 학습하고 원리를 이해해야 합니다.
- 추가적으로, 다른 팀과 차별화할 수 있는 확장 기능이나 개선 요소도 함께 고민해 봅니다.

## 3. 중점 포인트

- `WHERE id = ?` 조건에서 B+ 트리를 어떻게 활용할 것인지 설계합니다.
- 대용량 레코드를 효율적으로 생성하고 테스트할 수 있는 방안을 마련합니다.
- 이전 차수에서 만든 SQL 처리기와 B+ 트리 인덱스를 자연스럽게 연결합니다.

## 4. 품질 기준

- 단위 테스트를 통해 주요 함수를 검증합니다.
- 기능 테스트를 통해 B+ 트리가 올바르게 적용되는지 확인합니다.
- 엣지 케이스를 최대한 폭넓게 고려합니다.
- 이력서와 포트폴리오에 포함할 수 있을 만큼 완성도 높은 수준으로 구현합니다.

## 5. 결과물 발표

- 발표는 목요일 오전에 진행됩니다.
- 각 팀은 발표를 통해 프로젝트 결과를 공유합니다.
- 팀 프로젝트 결과물을 간단한 데모 형태로 시연합니다.
- 별도의 발표 자료는 만들지 않으며, GitHub 저장소의 `README.md`를 기준으로 설명합니다.
- AI를 활용한 빠른 구현만큼, 생성된 코드에 대한 철저한 검증과 테스트도 중요합니다.
- 반드시 테스트 케이스를 작성하고 검증 과정을 수행한 뒤, 그 내용을 데모 발표에도 포함해야 합니다.
- 발표 시간은 `4분`, Q&A 시간은 `1분`입니다.

## 6. 정리

이번 과제의 핵심은 B+ 트리 인덱스를 실제 SQL 처리 흐름에 연결하고, 대규모 데이터 환경에서 인덱스의 효과를 검증하는 것입니다. 빠른 구현도 중요하지만, 핵심 로직을 충분히 이해하고 설명할 수 있을 만큼 학습과 검증을 병행해야 합니다.
