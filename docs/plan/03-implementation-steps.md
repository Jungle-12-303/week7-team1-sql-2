# 03-implementation-steps

## 단계 1: 저장 계층 전환 준비
- 입력: 기존 INSERT/SELECT 경로
- 작업: 텍스트 저장 추상화 분리, 바이너리 reader/writer 인터페이스 추가
- 산출: 저장소 인터페이스 교체 가능한 상태
- 검증: 기존 테스트 회귀 통과

## 단계 2: 바이너리 쓰기/읽기 기본 구현
- 입력: 단계 1 결과
- 작업: 레코드 직렬화/역직렬화, row_ref(offset) 계산 로직 구현
- 산출: 바이너리 `.data` 파일 입출력 동작
- 검증: 저장 후 동일 값 복원 가능
- 주석 규칙: 바이너리 포맷과 길이 필드 처리 규칙을 한국어 주석으로 설명

## 단계 3: 텍스트 데이터 마이그레이션
- 입력: 단계 2 결과
- 작업: 기존 텍스트 `.data` -> 바이너리 `.data` 변환 스크립트/함수 추가
- 산출: 일회성 마이그레이션 경로
- 검증: 변환 전후 row 수/값 일치
- 주석 규칙: 변환 단계와 검증 기준을 한국어 주석으로 설명

## 단계 4: INSERT 자동 ID + 인덱스 등록
- 입력: 단계 3 결과
- 작업: `INSERT` 시 자동 ID 부여 후 바이너리 저장 + `index_insert`
- 산출: 삽입 경로 연동 완료
- 검증: 삽입 직후 id 조회 가능
- 주석 규칙: 자동 ID 정책과 인덱스 등록 순서를 한국어 주석으로 설명

## 단계 5: SELECT id 분기 + 안정화
- 입력: 단계 4 결과
- 작업: `WHERE id = ?` 패턴 판별, 인덱스 조회 분기, 예외/fallback 처리
- 산출: id 전용 fast path + 안정화된 에러 처리
- 검증: id 조건은 인덱스, 일반 조건은 선형 탐색 유지
- 주석 규칙: 분기 조건과 fallback 기준을 한국어 주석으로 설명

## 단계 6: 테스트 보강
- 입력: 단계 5 결과
- 작업: 단위 테스트 + 기능 테스트 + 마이그레이션 검증 테스트 추가
- 산출: 테스트 세트 업데이트
- 검증: 전체 테스트 통과

## 단계 7: 벤치마크 + 문서 마감
- 입력: 단계 6 결과
- 작업: 100만 건 삽입, 인덱스/선형 성능 비교, 텍스트 대비 바이너리 개선율 기록, README 반영
- 산출: 측정 결과와 데모 절차 문서화
- 검증: README만으로 데모 가능

## 병렬 가능 작업
- 병렬 A: 테스트 데이터 생성 스크립트 준비
- 병렬 B: 텍스트->바이너리 마이그레이션 검증 도구 준비
- 병렬 C: README 데모 섹션 초안 작성
- 병렬 D: 성능 측정 포맷(표/로그) 템플릿 작성
