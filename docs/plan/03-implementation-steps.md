# 03-implementation-steps

## 단계 1: 기초 구조 준비
- 입력: 기존 INSERT/SELECT 경로
- 작업: ID 카운터/인덱스 초기화 지점 추가
- 산출: 초기화 코드와 빌드 통과
- 검증: 기존 테스트 회귀 통과

## 단계 2: INSERT 자동 ID + 인덱스 등록
- 입력: 단계 1 결과
- 작업: `INSERT` 시 자동 ID 부여 후 저장 + `index_insert`
- 산출: 삽입 경로 연동 완료
- 검증: 삽입 직후 id 조회 가능
- 주석 규칙: 자동 ID 정책과 인덱스 등록 순서를 한국어 주석으로 설명

## 단계 3: SELECT id 분기 추가
- 입력: 단계 2 결과
- 작업: `WHERE id = ?` 패턴 판별 및 인덱스 조회 경로 추가
- 산출: id 전용 fast path
- 검증: id 조건은 인덱스, 일반 조건은 선형 탐색 유지
- 주석 규칙: 분기 조건과 fallback 기준을 한국어 주석으로 설명

## 단계 4: 예외 처리/안정화
- 입력: 단계 3 결과
- 작업: 인덱스 실패/미존재/파싱 오류 처리
- 산출: fallback 및 에러 코드 정리
- 검증: 엣지 케이스 테스트 통과
- 주석 규칙: 실패 케이스 처리 의도를 한국어 주석으로 설명

## 단계 5: 테스트 보강
- 입력: 단계 4 결과
- 작업: 단위 테스트 + 기능 테스트 추가
- 산출: 테스트 세트 업데이트
- 검증: 전체 테스트 통과

## 단계 6: 벤치마크 + 문서 마감
- 입력: 단계 5 결과
- 작업: 100만 건 삽입, 인덱스/선형 성능 비교, README 반영
- 산출: 측정 결과와 데모 절차 문서화
- 검증: README만으로 데모 가능

## 병렬 가능 작업
- 병렬 A: 테스트 데이터 생성 스크립트 준비
- 병렬 B: README 데모 섹션 초안 작성
- 병렬 C: 성능 측정 포맷(표/로그) 템플릿 작성
