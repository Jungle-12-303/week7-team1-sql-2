## 단위 테스트
- 테스트 1: `next_id`가 증가 순서를 보장한다.
- 테스트 2: `index_insert` 후 `index_find`가 동일 row_ref를 반환한다.
- 테스트 3: 없는 id 조회 시 빈 결과를 반환한다.
- 테스트 4: id 중복 삽입이 방지된다.
- 테스트 5: id 조건 판별 함수가 정확히 동작한다.
- 테스트 6: 바이너리 직렬화/역직렬화 후 원본 값이 보존된다.
- 테스트 7: row_ref(offset)로 읽은 레코드가 기대 값과 일치한다.

## 기능 테스트
- 시나리오 1: 다건 INSERT 후 `WHERE id = ?` 정확 조회
- 시나리오 2: `WHERE name = ?`는 기존 선형 탐색으로 정상 동작
- 시나리오 3: 인덱스 미스/파싱 실패 시 fallback 동작 확인
- 시나리오 4: 텍스트 데이터 -> 바이너리 마이그레이션 후 결과 일치
- 시나리오 5: `WHERE id >= ?`, `WHERE id < ?` 범위 조회가 인덱스 경로로 동작
